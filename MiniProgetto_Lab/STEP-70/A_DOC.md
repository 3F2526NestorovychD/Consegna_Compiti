## Attivita elementari di sviluppo

Quanto segue non e la metodologia o una metodologia di programmazione, sono le attivita di base che, generalmente, in diversa misura, ricorrono in tutti gli sviluppi software.

Le vere metodologie, che al livello di base non sono ancora necessarie, definiscono in dettaglio approcci specifici per tali attivita. Per ora ci basta seguire questo schema.

| Attivita | Descrizione (ancora da compilare) |
| --- | --- |
| Requisiti e specifiche | Che cosa deve fare il software? [INSERIRE] |
| Analisi | Hai compreso correttamente il testo dell'esercizio? Note e chiarimenti: [INSERIRE] |
| Architettura | Serve una struttura di alto livello? Se si, descriverla. Se no, indicare non necessaria. [INSERIRE] |
| Algoritmo | Descrivi il processo di soluzione o implementazione. Scegli una forma: flowchart, pseudo-codice, UML. [INSERIRE] |
| Progettazione | Come traduci l'algoritmo in componenti/strutture? [INSERIRE] |
| Programmazione | Linguaggio, strumenti, note di implementazione. [INSERIRE] |
| Testing | Dati di input, casi di test, risultati attesi. [INSERIRE] |
