## Attività elementari di sviluppo

Quanto segue non è "la metodologia" o "una metodologia di programmazione", sono le attività di base che generalmente, in diversa misura, ricorrono in tutti gli sviluppi software.

Le "vere" metodologie, che al livello di base non sono ancora necessarie, definiscono in dettaglio approcci specifici per tali attività. Per ora ci basta seguire questo schema.

| Attività | Descrizione |
| --- | --- |
| Requisiti e specifiche | Descrizione, chiara e completa, di ciò che il software deve fare.<br>Nel mondo reale possono richiedere centinaia o migliaia di pagine scritte da molti esperti di alto livello.<br>Nel nostro caso: si tratta del testo degli esercizi del libro o definiti dall'insegnante. |
| Analisi | Analisi dei requisiti e specifiche per essere sicuri di averli compresi correttamente, a volte si cominciano ad abbozzare soluzioni, a livello molto alto.<br>Nel mondo reale è un task importante che produce documenti.<br>Nel nostro caso: normalmente si tratterà di assicurarsi di aver compreso correttamente e completamente il testo dell'esercizio da svolgere, facendo domande al docente se necessario.<br>In alcuni casi (progetti) sarà utile produrre un documento informale ma preciso. |
| Architettura | Ha senso per veri sistemi software.<br>Nel nostro caso normalmente non sarà necessaria. |
| Algoritmo | Identificazione del processo di soluzione o implementazione, indipendente dal linguaggio.<br>Nel nostro caso uno dei seguenti:<br>- Diagramma di flusso prodotto con Flowgorithm per semplici programmi procedurali<br>- pseudo-codice<br>- UML (Object Oriented e alcuni casi specifici) |
| Progettazione | La definizione dei meccanismi di soluzione/implementazione è influenzata dalle caratteristiche degli strumenti di implementazione.<br>Per semplici esercizi e programmazione procedurale l'identificazione dell'algoritmo coinciderà con la progettazione.<br>Per casi non banali si potrà utilizzare notazioni dedicate come UML. |
| Programmazione | Codifica, della progettazione, nel linguaggio di programmazione. |
| Testing | Nel nostro caso creazione, senza tools dedicati, di alcuni dati di input (10-100).<br>Si consiglia di creare una normale "funzione" dedicata alla generazione di tali dati e una per eseguire i tests. |
